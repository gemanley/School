There is a level-sensitive event control called wait. This is written as
wait (expression)
When the program counter reaches this event control, the expression is evaluated, and, if false, then execution is suspended until the expression becomes true. If the expression is true when the statement is reached, then the wait has no effect, and execution proceeds to the controlled statement.

This event control is a convenience, since you can achieve the same effect using a combination of the other controls. For example,

wait (f(x, y, z)) s1;

is equivalent to

while (!f(x, y, z)) @(x or y or z) ;
s1;

Again, the "while" statement is explained under Loops later in this chapter.
