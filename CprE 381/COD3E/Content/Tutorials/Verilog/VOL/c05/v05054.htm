We saw an example of a race condition before where a two-stage pipeline was coded as: 

always @(posedge clock)         always @(posedge clock)
    dff1 = f(x);                    dff2 = dff1;

As noted before, this doesn't work because dff1 is being both sampled and changed at the same instant of simulated time in different processes. Think about how you would solve this problem. Then, click each Solution button to see sample solutions.
