When the event expression consists of a net or register, any value change on that net or register satisfies the event. In the example above, @state would be satisfied whenever state changed value. This could happen for any number of reasons, most commonly continuous or procedural assignment, but including task parameter update and port assignment. Note that x->z and z->x are considered value changes, as are 1->x, 0->x, x->1 and x->0.

In this way, simulation is somewhat different from real life, since the x in simulation is a 1 or 0 in real life. Consequently, the simulation may indicate a change event that the real circuit might not see.
