The edge events are extremely useful. They are defined as:

posedge negedge
0,x,z->1 1,x,z->0

Edge events are useful for modelling clocked logic elements, like flip-flops. They are also useful for synchronizing activity in a model based on a common clock. For example, in the following two always blocks, 

	always @(posedge clock)		always @(negedge clock)
		x = f(y);			z = x;
the value of z will always be assigned the previously computed value of x.
