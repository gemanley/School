Casex and casez are variations of case which allow for "don't care" bits in the comparison operations. Depending on the keyword, if a bit is z (casez) or x or z (casex) in either the selector or the item expression, then that bit does not participate in the comparison. Since the expression values are determined at simulation time, the don't care bits can be dynamic.

As a convenience for specifying constants used in item expressions, the ? character can be used in constants to indicate a z value (not x!).

Example

     casez (sel)
             4'b0000:        case0;  // task named case0
             4'b0001:        case1;
             4'b?1??:        case2;
             4'b10??:        case3;
     endcase
In the above example, execution would yield the following, depending on sel

sel
 		 task
executed
0000
0001
0100, 0101, 0110, etc.
1000, 1001, 1010, etc.		 case0
case1
case2
case3

In the above example, the same result would be achieved with casex. However, casex can be used for dynamic masking in ways that casez cannot, since operations with z bits in operands generally turn into x bits in the results.
