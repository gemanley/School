Part-select is the Verilog term for accessing a subset of bits from a vector. The set of bits accessed must be contiguous, and again this operation may be done for either obtaining or modifying the set of bits.

Part-select bounds must always be constant-valued expressions, regardless of whether this is a left-hand side or a right-hand side access. Part-select bounds must also follow the declaration of the vector. That is, the following is incorrect

reg [1:20] vbad;
...
x = vbad[3:2];

because the part-select bounds (3:2) are the wrong way around.

If the part-select range is out of range of the vector, then the result is x. If the part-select range is partially out of range, then at least the out-of-range bits of the result will be x. Whether any of the bits will be valid data bits is implementation-dependent. The result is also x if either part-select bound is x. Because part-select bounds must be constants, this is a pretty unusual case. As with bit-select, it is illegal to apply part-select to a real register.
