The process of debugging a Verilog model usually follows the following stages. Click each stage to learn more about it.

1. Edit the Source
2. Compile the Model
3. Run the Simulation Until the First Error Occurs
4. Debug the Problem
5. Repeat the Process

Once the simulation exposes an error, the problem must be debugged. Once an error occurs, simulation may stop immediately due to internal checking, or the simulation may run for a long time before you gain control. If the simulation was run non-interactively, you must simply rely on whatever output may have been produced by the simulation to debug the problem.
