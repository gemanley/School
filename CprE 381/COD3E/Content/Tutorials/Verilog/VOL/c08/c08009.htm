The process of debugging a Verilog model usually follows the following stages. Click each stage to learn more about it.

1. Edit the Source
2. Compile the Model
3. Run the Simulation Until the First Error Occurs
4. Debug the Problem
5. Repeat the Process

After compilation, the simulation begins. This is under user control, as you can use a command-line flag (-s) with Verilog-XL to gain control before simulation begins. With compiled simulators, like VCS, running the simulation is a separate command altogether, and the same command-line flag will allow you to gain control before simulation begins.
