A neat trick to get around this problem is to include the $dumpvars (or equivalent) call, but to only execute it under program control. If the model is then checkpointed periodically (using $save), the model can be restarted from the checkpoint with the dump control command line argument turned on.

For example, the checkpoint/restart code could look like this:

        always #100000
            begin
                $save("checkpoint.file");
                if ($test$plusargs("dump")) $dumpvars;
            end

This would allow you to start the last checkpoint file and use "+dump" on the command line.
