To make it easier to cover all the input cases, the ? character can be used in an input field of the table. This means the iteration of 0, 1, and x, so a line which was

      0 ? : 0;

would be equivalent to

      0 0 : 0;
      0 1 : 0;
      0 x : 0;

So, the following table would yield identical behavior to the built-in and:

   table
   // in1 in2 : out
      0 ? : 0;
      ? 0 : 0;
      1 1 : 1;
   endtable
