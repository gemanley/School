When a complex expression is evaluated, the context of each operation is the entire expression. So, in the following expression,

s16 = (r16 + p16) >> 1;

the context is 16, L(r16) = L(p16) = 16, so the addition is done in 16 bits. Consequently, s16[15] will always be 0. This is probably not what you intend.

The solution to this is to modify the expression so that either one of the operands is at least 17 bits long, or to modify the context. Both of the following will work:

s17 = (r16 + p16) >> 1;
s16 = (r16 + p16 + 17'd0) >> 1;
