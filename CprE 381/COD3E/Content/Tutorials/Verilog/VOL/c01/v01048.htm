Signal names in the instance port list are matched up left-to-right with signal names in the module definition port list. If you want to omit a port in the instance list, simply use consecutive commas. E.g.,

          foo f1 (source1, , sink1, , bus);

would leave the second input port and the second output ports unconnected. Verilog imposes no restrictions on the order of ports in a port list in the definition. That is, inputs, outputs, and inouts can be mixed in any order in the definition. In the instantiation, however, the order must match the definition. Any number can be omitted in the instantiation. It does not make sense, however, to omit a port in the module definition.
