Another type of race occurs when the model behavior depends on the number of events which occur at a given simulation time. Consider this example: 
    always @(ris1 or rp2)
        evcount = evcount + 1;
        
    assign ris1 = r == 1;
    assign rp2 = r + 2;
    
    initial begin
        evcount = 0;
    #10 r = 1;
    #1  $display(evcount);
    end
In this code, when r changes at 10, the always @(ris1 or rp2) will be triggered either one or two times, depending on the order of evaluation and propagation of the two continuous assignments. When this code executes, evcount will have a value of either 1 or 2.
