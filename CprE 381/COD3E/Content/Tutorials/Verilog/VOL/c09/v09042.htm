The utility routines which are provided allow you to obtain data from the model and let you modify data in the model. There are too many utility routines (more than 200 at last count) to describe them all here. We will describe some of the most common, and use some in examples. You should consult the Verilog-1364 LRM for complete descriptions.

In general, there are two classes of utility routines:

1. TF routines
Routines whose names begin with "tf_". These routines are used when the signal (net or register) to be read or written (register) has been identified in the system task or function's parameter list.

2. ACC routines
Routines whose names begin with "acc_". These routines can be used to access arbitrary data values within the model.

There is history involved in the two sets of routines. The TF routines were the first implemented, and are more limited than the ACC routines. Consequently, they are simpler and more efficient. The ACC routines, on the other hand, can do many more things than the TF routines can.
