Homework on functional dependencies  

1. You are given the following relation r over ABCD.

 

B -> ACD and C -> D.

r
A B C D
6 2 3 4
1 3 8 5
6 7 8 5

Recall that the term "key" without any qualification stands for unique key that is required to be minimal in the sense that if any attribute in the key is removed, the remaining attributes no longer form a key. Also, recall that a key can contain multiple attributes and that a relation can have more than one key.

(A) What are all the keys in this relation?

(B) Give example of a superkey that is not a key.

(C) Give example of a superkey that is also a key.

(D) Give example of a set of attributes that is not a key. Prove your answer.

(E) Give the most obvious (no brainer) example of a superkey.

(F) Give example of a trivial functional dependency.

(G) Give example of a non-trivial functional dependency over ABCD that is satisfied by the given state of relation r. Prove your answer.

(H) Give example of a functional dependency over ABCD that is not satisfied by r. Prove your answer. 

2. Suppose we have a database containing a single relation r over ABCD that is required to satisfy the functional dependencies AB ® C, C ® D. Give example of functional dependency that is satisfied in some state of relation r, but does not hold in r.

3. Deduce AC -> E from A -> B, C -> D, and BD -> E.

4. Show that the following is not a valid rule for deduction of functional dependencies.

AB -> C, B -> C, then A -> C

5. Prove that {A -> B, B -> C, C-> A}+ = {A -> B, B -> A, B -> C, C -> B}+.

6. Given functional dependencies A -> B, BC -> D.

(A) Prove that the decomposition (AB, BCD) is lossy.

(B) Prove (ABC, BCD) is lossless.

7. Given a database with attributes A, B, C and D where the functional dependencies AB -> C, C -> D, and D -> A are supposed to hold.

(A) Give a BCNF decomposition. 

(B) Can all the functional dependencies be enforced in your design in (A).
