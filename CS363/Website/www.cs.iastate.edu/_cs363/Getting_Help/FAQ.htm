Q7: How to set up the path and classpath for running java program? How to compile and run a java program?

In Com S 363, we highly recommend that you streamline all these through the use of a Windows batch file. Sample command files are provided with each project. Here are some tips:

A. The following command sets path variable. It tells Windows where to find java executables. Note the underscore character carefully.

set PATH=c:\j2sdk1.4.0_01\bin\

B. The following command sets CLASSPATH variable.  It tells java where to find the class files and the libraries in order to execute java programs. Make sure the underline part is the directory in which your java lib is located. Also note that "." tells java to look for a class in the current directory (where the batch file is being executed from).

set CLASSPATH=.; c:\J2SDK1.4.0_01\lib

C. To compile a java program that is in MyProg.java the following command is needed.

javac MyProg.java

When the above command is executes, java will create MyProg.class. Java has its own way of executing a class file. Java considers a class file as an executable program.

FYI: The code contained in a java program is called java byte-code. The byte-code is meant to be machine independent and can be executed on java virtual machine (JVM). JVM is an executable, included with java for every platform.

D. The following will compile all java programs in the current directory (where the batch file is being executed from).

javac *.java

E. The following will run MyProg.class

java MyProg 

If MyProg outputs something to the screen, the output can be redirected to a file, say MyFile.txt by the following command.

java MyProg > MyFile.txt

Then MyProg is executed, the file MyFile.txt will be created in the current directory (where the batch file is being executed from). If the file has to be stored elsewhere, full path name should be provided (e.g. java MyProg > D:\MyFavoriteFiles\PyFile.txt).

Q8: What is wrong if I get the error message saying “'javac' is not recognized as an internal or external command, operable program or batch file.”?

If you get such an error, that means your java path is not set up correctly. The system couldn’t find java.exe. So please set up the path for java. If you already have a path command in your batch file, make sure that it is correct.

Q9: How to check the PATH and CLASSPATH system variables are already set up?

In your project directory, create a text file called Check.bat with the following command in it.

set

Now, double click on Check.bat. This will list all the system variables including path and classpath. You can check if your path and classpath are on the list. If they are, you set up them correctly. Otherwise, you have to set up again.

Q3: Why can't I get EXCEPT, NJOIN, INTERSECT to work in SQL Runner. What operators are not supported or only partially suported. 

If some operators are not working in SQL Runner, most likely the problem is not with the Runner. The Runner does not really execute any commands, it just sends them for execution to the database system being connected to. In our course we use the built-in database capabilities of the XP + .NET Framework that are at par with Microsoft Access. Although SQL is supposed to be an industry standard, in reality there are differences across database platforms. If you connected to a different database system some commands that do not work may work there and vice versa. 

Here is a list of commands that you may experience some problems with.

  • EXCEPT is not supported
  • INTERSECTION is not supported
  • NATURAL JOIN (or NJOIN) is not supported.
  • Union is supported in the form "(select ... ) union (select ...)" but not in a nested form such as "select * from (... UNION ...)".
  • IN and NOT IN are supported, but they can be used only for one attribute. For example, the following usage is supported: ... x.A in (select y.B from ...), but the following is not supported: "... (x.A, x.B) IN (select y.C, y.D from ....)".

FAQ on Project 3

Q1: I get an error when I execute an XQuery with the return clause containing two or more elements.

You cannot return multiple elements. Examples of incorrect and correct versions are shown below:

The following is not correct:

... return $i/Rank, $i/Salary

The following is correct:

... return <RankAndSalary>
              $i/Rank,
              $i/Salary
           </RankAndSalary>

Q2: What is the syntax of arithmetic comparison operators?

Our runner currently uses Kweelt to execute XQuery queries. Arithmetic comparison operators have to be escaped inside Kweelt. We offer for each operator two alternatives as defined in the table below:

Operator Alternatives
< use LT or escape as .<.
<= Use LEQ or escape as .<=.
> Use GT or escape as .>.
>= Use GEQ or escape as .>=.

 

 

 

 

Q3: For Item 04, what do you mean by "make sure that there is one element per student"?

"One element per student" means that all information to be reported for a given student should be put inside a single element. For example, suppose a student with id s55 is enrolled in CS301, CS 320, and CS 415 can be reported as follows. Here an "Entry" tag is used to build a single element for all the information to be recorded.

<Entry>
    <StudentID>s55</StudentID>
    <CourseCode>CS301</CourseCode>
    <CourseCode>CS320</CourseCode>
    <CourseCode>CS415</CourseCode>
</Entry>

Note that all students will not be taking the same number of courses. Those students who are not enrolled in any course should not be reported.

FAQ on Project 4:

Q1: Why do we need to use the iterator? I can write the program without using an iterator.

You MUST try to understand that a profoundly important point is being made here.   

Iterators are important for container classes. Collections are important container classes and they arise naturally in databases. For example, a relation is a set of tuples, an xml document is a sequence of elements. Iterators are designed to supply one item (tuple, element, etc.) at a time as needed by a client hiding the drudgery of low-level navigation within a collection.

In this project you are wearing two hats. The application for xml document that you are writing is client of the iterator, The application has its own logic. You are also implementing the iterator, that has its own logic requiring you to get into cumbersome internal navigation of the xml document. The two logics are indeed independent of each other. Writing a program means without the iterator will absolutely lead to spaghetti-code. Once you have implemented an iterator, you can use it to solve a variety of applications requiring you to concentrate only upon the logic of the application.

The concept of JDBC-like programming paradigm is based upon the concept of an iterator. At the heart of JDBC is the result-set iterator. For relational databases iterators are simple, because all objects (tuples) are uniformly structured. For xml, iterators can be significantly complex and a framework such as JDBC is not yet available.

Iterators are also very important in database implementation. Without iterators, automatic generation of object-code would be very difficult, prone to bugs, and very difficult to maintain. This because the clients view of a relation as a set of tuples is very distant from system's block oriented view of tuples.

If you are still having difficulty understanding this point or are more curious, you are encouraged to see the instructor. He will show you how iterators are composed to automate code generation for database queries that is composed of subqueries.

Q2: I got an error about the type "double" when doing xtp.getElement().

xtp.getElement() will return a String. You have to first convert the String to a double, then assign the double value to the GPA variable. The function of converting String to double is: Double.parseDouble() which takes a String argument and returns a double data type.
