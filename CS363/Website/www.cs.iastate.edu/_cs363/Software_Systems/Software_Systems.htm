A. CyDIW 

CyDIW is a seamless low profile GUI-based multi-purpose develop environment for experimenting with multiple command-based systems. It can be used to connect to multiple database servers and our own database prototypes and use them simultaneously with great ease. Batches of commands from multiple systems can be developed and executed.

Install CyDIW. Obtain CyDIW from Resources/CyDIW_Root.zip and unzip it in a folder of your choice. (On computers in Pearson store it in a folder on U: drive.) That is all what is needed to install it.

Run a demo in CyDIW (Windows users). Double click on StartCyDIW.bat. A GUI window will open. As stated above, the GUI is provides an integrated development environment (IDE) for command-based systems. It has an editor - cum - launchpad  for batches of commands that are stored as plain text files. Use the [Open] button to load ComS363_XQuery(Saxon)_Demo.txt file. (This file is in the ComS363 folder created earlier.) This is a demo for XQuery that uses the Saxon XQuery query engine pre-packaged and configured in CyDIW for immediate use. Click [Run All] button. You should see the output in the output pane. Save this output in the ComS363_XQuery(Saxon)_Demo.htm file in a folder P0.

Non-Windows users. Instead of StartCyDIW.bat, execute startcydiw.sh instead in a Unix-based system. Caution: CyDIW is not stress tested for non-Windows platforms. If you have problems, we will work with you to some extent, but any feedback will especially be useful in future. If problems persist, you may consider switching to Windows.

FYI - What is CyDIW. Any command-based system can be configured and used with CyDIW (Cyclone Database Implementation Workbench). More significantly, it also includes infrastructure for implementation of database internals. This infrastructure is used by several database prototype platforms that we are implementing. The platform offers a good paradigm for database implementation leading to very high level encapsulation in terms of batches of commands. CyDIW can be used by students, TAs, instructors, developers, and project directors. Get this if you want to learn more; a knowledge of this is not required for the course.

Difficulties with CyDIW and suggestions. Please keep us posted and discuss them with us. CyDIW includes numerous suggestions made by students in past. We are very keen on making improvements.

C. Configuration of CyDIW for MySQL

CyDIW is preconfigured for MySQL - a popular quasi Open Source SQL database engine. A database named xyzDB - where xyz is your ISU NetID - has been created for your individual use on our MySQL server in Pearson. CyDIW needs to be configured so that it can connect to your database.

Configure CyDIW for your database. First close CyDIW if it is running. Open SystemConfig.xml file for editing. Make sure to use a plain text or XML editor that will not leave any hidden characters in the file. The following shows the configuration setting for MySQL. Carefully replace highlighted portions with your information provided below. Be careful that the file is saved as a plain text file

 <Client Name="MySQL" Prefix="$MySQL" Enabled="yes"
      ClassPath=""
      LibraryPath="sql\drivers"
      ClientAdapter="clientsmanager.clients.SQLClientAdapter" DriverClass="com.mysql.jdbc.Driver"
      ConnectionString="jdbc:mysql://csdb.cs.iastate.edu:3306/NameOfYourDatabase"
      Username="YourUserName" Password="YourPassword" />

For example, if your ISU Net ID were “lexluthor” and your Student ID number was “601624656”, you would use:

  • YourUsername is your ISU Net ID (lexluthor)

  • YourPassword is your ISU Net ID + "-" + Last 2 of Student ID Number (lexluthor-56)

  • NameOfYourDatabase is your ISU Net ID + "DB" (lexluthorDB)

Note that as a database lexluthorDB is empty. In this database you have the privilege to create tables, populate them, query and update them, and delete them. This is all what matters for the project. You cannot delete the database. The password we have provided is a low security issue. By using your username and password, others can perhaps look at your database but they cannot see your code (the batch of commands) that was used to arrive at the current state of your database.

Run a demo. Double click on StartCyDIW.bat to start CyDIW. Using the [Open] button load the batch of commands in ComS363_MySQL_Demo.txt. (This file is in the ComS363 folder created earlier.) Then press [Run All] button. The demo should run. Save the contents of the Output Pane in ComS363_MySQL_Demo_Output.htm file in folder P0. Be sure to provide the .htm extension as a part of your file name. You will submit this file as a part of Project P0. 

D. Setup JDBC and run a demo 

Setup JDBC. Download and store MySQL driver mysql-connector-java-5.0.5.jar and the demo program JDBC_Demo.java in the same working directory. Open the java file for editing and replace NameOfYourDatabase, YourUserName, and YourPassword" with the same values as in Part C. above.

Setup the database. Run the MySQL demo, covered in Section C above, so that your database has two tables with tuples in them.

Expected output of the demo. To see the expected output of the demo please click here. 

Option A. Setup and running the demo for Eclipse users. Follow the steps below. (i) Create a project congaing JDBC_Demo.java. (ii) Add the MySQL driver to your build path. To see it in a picture, click here. (iii) Run the demo program. 

Option B. Running the demo for command line users. The commands are as follows:

javac JDBC_Demo.java
java -cp ".;mysql-connector-java-5.0.5.jar;" JDBC_Demo > output.txt

Option C. Running the demo using a batch file. Download JDBC_Demo.bat.txt and store it in the working folders with other files. Edit the path and class path variables in the same manner as Option C in Section C. After saving the .bat file and removing the ".txt" suffix from its name, double click on it to run. The file JDBC_Demo_Output.txt congaing the expected output will be created. Store this file in folder P0.
